🛡️ Step-by-Step OWASP ZAP Scan Guide & Automation

[ Show Guide ]

OWASP ZAP (Zed Attack Proxy) performs Dynamic Application Security Testing (DAST) by probing the running application from the outside, validating all vulnerability vectors listed in the project's penetration testing checklist.

  1. Install OWASP ZAP Desktop: Download ZAP for Windows from zaproxy.org/download and install it.
  2. Run the Target Application: Open a separate terminal and start your Laravel development server: php artisan serve Ensure this runs on http://localhost:8000 or http://127.0.0.1:8000.
  3. Configure the Automated Scan:
    • Open OWASP ZAP Desktop.
    • Click on the Quick Start tab in the middle pane.
    • Click the big Automated Scan button.
    • Set URL to attack to: http://127.0.0.1:8000
  4. Launch Attack: Click the Attack button. ZAP will run a spider crawl followed by active dynamic testing. This simulates real attacks (XSS, SQL Injection, IDOR, Traversal, etc.).
  5. Generate & Save JSON Report:
    • Wait for both progress bars at the bottom to reach 100%.
    • Go to the top menu and select: Report > Generate Report.
    • Under **Template**, select JSON Report (this is crucial for parsing).
    • Click Generate Report and save the file directly in the RIKMS root folder as: zap-report.json
  6. Visualize: Click the Refresh Dashboard button at the top of this dashboard page.
🤖 Dashboard Scan Automation:
This dashboard script has been automated to search for standard ZAP installations on your computer (in standard Program Files paths or system PATH) and trigger a headlessly automated quick scan against the Laravel app if it detects it running on port 8000 or 8080! Check the python terminal log for scan progress.
